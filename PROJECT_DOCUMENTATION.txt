PROJECT OVERVIEW
================================================================================

HealthSaathi is an AI-powered health companion platform designed to provide 
instant medical triage, nearby health facility discovery, and personalized 
medical guidance. The system is built with a multilingual interface (Nepali 
and English) and serves users across 77 districts in Nepal.

Core Features:
  • AI-powered symptom triage and risk assessment
  • Facility location discovery using GPS mapping
  • Multilingual support (Nepali and English)
  • Conversation history management
  • Text-to-Speech health guidance
  • User authentication and profile management
  • Real-time medical consultation support


TECHNOLOGY STACK
================================================================================

BACKEND:
  • Framework: Django 4.2.29 (Python web framework)
  • REST API: Django REST Framework 3.16.1
  • Database: PostgreSQL (via psycopg2-binary)
  • Vector Database: ChromaDB 1.5.5 (for AI embeddings and similarity search)
  • Mapping API: Baato 1.0.1 (location and facility mapping)
  • Server: Gunicorn 20.1.0 (production WSGI server)
  • CORS: django-cors-headers 4.9.0 (cross-origin requests)
  • Environment: python-dotenv 1.2.2 (environment configuration)
  • HTTP Client: requests 2.32.5 (external API calls)

FRONTEND:
  • UI Framework: React 18.2.0 (modern component-based UI)
  • Build Tool: Vite 7.3.1 (fast module bundler)
  • Styling: Tailwind CSS 3.3.5 (utility-first CSS framework)
  • Routing: React Router DOM 6.20.0 (client-side navigation)
  • HTTP Client: Axios 1.6.0 (API communication)
  • Mapping: Leaflet 1.9.4 + React Leaflet 4.2.1 (interactive maps)
  • CSS Processing: PostCSS 8.4.31 + Autoprefixer 10.4.16

INFRASTRUCTURE:
  • Version Control: Git
  • Development: Node.js ecosystem for frontend

CODE FLOW AND ARCHITECTURE
================================================================================

USER AUTHENTICATION FLOW:
1. User accesses LoginPage.jsx
2. Enters email and password credentials
3. Frontend calls triageService.login() → api.post('/login/')
4. Backend LoginView validates credentials against User model
5. Django generates authentication token
6. Frontend stores token in localStorage
7. User redirected to ChatPage.jsx
8. AppContext maintains authenticated user state globally

SYMPTOM TRIAGE FLOW:
1. User enters symptoms in ChatPage.jsx
2. Frontend calls triageService.sendTriage(symptoms, location)
3. API request sent to backend TriageView with:
   - User symptoms (text)
   - User location (latitude, longitude)
   - User context from profile
4. Backend TriageView processes:
   - Validates user authentication
   - Creates TriageSession record in database
   - Sends symptoms to AI model (ai_client.py)
   - AI analyzes and generates:
     * Risk level (HIGH/MEDIUM/LOW)
     * Brief medical advice
     * Detailed health guidance
     * Dietary recommendations (food_eat, food_avoid)
     * Dos and Don'ts
   - Uses ChromaDB for semantic search of similar symptoms
   - Returns structured JSON response
5. Frontend receives response and displays RiskCard
6. User sees:
   - Risk assessment with color coding
   - Medical advice in current language
   - Facility recommendations
   - Text-to-speech playback option

LOCATION-BASED FACILITY DISCOVERY:
1. ChatPage requests user location via locationService.getCurrentLocation()
2. Browser Geolocation API returns latitude/longitude
3. locationService.getPlaceAndCity() calls backend location endpoint
4. Backend uses Baato API to:
   - Reverse geocode coordinates → place name and district
   - Fetch nearby health facilities (hospitals, clinics)
5. Frontend displays:
   - Current location with district name
   - Interactive map with facility markers
   - Facility details (name, distance, type)
6. Facilities specific to user's district shown in triage advice


WORKING FLOW (USER PERSPECTIVE)
================================================================================

STEP 1: REGISTRATION
  • User opens HealthSaathi web application
  • Clicks "Create account" on LoginPage
  • Enters email, password, name, age, gender
  • Frontend sends to RegisterView endpoint
  • Backend validates and creates User record with UserProfile
  • User redirected to login

STEP 2: LOGIN
  • User enters email and password
  • Credentials validated by Django authentication
  • Token generated and sent to frontend
  • Token stored in browser localStorage
  • User taken to DashboardPage

STEP 3: LOCATION SETUP
  • Frontend requests browser location permission
  • GPS coordinates obtained via Geolocation API
  • Location reverse-geocoded to district and place
  • Displayed in location bar with refresh option

STEP 4: SYMPTOM INPUT
  • User navigates to ChatPage
  • Describes symptoms in chat input box
  • Can use voice input (VoiceButton) for hands-free entry
  • Can select from quick symptom suggestions
  • Sends message to AI triage system

STEP 5: AI ANALYSIS
  • Symptoms analyzed by AI model
  • ChromaDB searches for similar symptom patterns
  • System generates personalized health assessment
  • Risk level determined (HIGH/MEDIUM/LOW)
  • Medical advice prepared in user's language

STEP 6: RESULT DISPLAY
  • RiskCard shows:
    - Risk indicator with color and risk level
    - Primary medical advice
    - Detailed guidance
    - Food recommendations
    - Dos and Don'ts
    - Recommended local facilities
  • User can:
    - Listen to advice via text-to-speech
    - Adjust TTS settings (speed, language)
    - View on interactive map

STEP 7: CONSULTATION HISTORY
  • Conversations saved to database automatically
  • User can view previous consultations
  • History filterable by date and risk level
  • Can delete old consultations
  • Useful for tracking health patterns

STEP 8: FACILITY DISCOVERY AND MAPPING
  • User can navigate to MapPage for nearest hospitals/clinics
  • Map displays user's current location (blue marker)
  • Color-coded facility markers:
    - Red: Hospitals/Health Clinics
    - Blue: User
  • User can:
    - Tap facility marker for details
    - View facility address and contact info
    - Calculate distance to facility
    - Plan route to facility
  - Facilities sourced in real-time via Baato API
  - Filter facilities by type and distance

STEP 9: HEALTH RECORD TRACKING (DASHBOARD)
  • User navigates to DashboardPage for health analytics
  • Sees statistics about consultations:
    - Total consultations (count)
    - Consultation frequency (weekly, monthly)
    - Most common symptoms
    - Risk level distribution (how many high/medium/low)
  • Visualized through charts and statistics cards:
    - Trend graphs showing health patterns over time
    - Seasonal patterns in symptoms
    - Most-visited health facilities
  • Useful for:
    - Identifying chronic health patterns
    - Planning preventive healthcare
    - Tracking health improvements
    - Sharing data with doctors

STEP 10: PROFILE MANAGEMENT
  • User accesses ProfilePage for account management
  • Multiple tabs for different settings:
    * Personal Info: Update name, age, gender, contact details
    * Health Tab: Medical history, chronic conditions, medications, allergies
    * Account Tab: Email, password, security settings
    * Settings Tab: Language, notifications, voice preferences
  • Users can:
    - Update emergency contact information
    - Track medical history and chronic conditions
    - Export health records for doctor visits
    - Change language preference (affects translations globally)
    - Configure text-to-speech settings
    - Manage notification preferences
  • Profile data stored securely in PostgreSQL
  • Changes synced across all devices


DATA FLOW IN APPLICATION
================================================================================

REQUEST-RESPONSE CYCLE:

Frontend → Backend:
  /api/triage/
    Input: { symptoms, location, user_id, language }
    ↓
Backend Processing:
  1. Authenticate user via token
  2. Store TriageSession record
  3. Parse symptoms with NLP
  4. Query ChromaDB for embeddings
  5. Call AI model for analysis
  6. Query Baato API for facilities
  7. Compile response
    ↓
Backend → Frontend:
  Output: {
    risk_level, brief_advice, detailed_advice,
    food_eat, food_avoid, dos, donts,
    recommended_facilities, conversation_id
  }
    ↓
Frontend Display:
  1. Parse response
  2. Render RiskCard component
  3. Update chat history
  4. Display on map
  5. Enable text-to-speech


API ENDPOINTS SUMMARY
================================================================================

AUTHENTICATION:
  POST /api/register/       - Create new user account
  POST /api/login/          - User login and token generation
  POST /api/logout/         - Logout and session cleanup
  GET  /api/profile/        - Fetch user profile
  PUT  /api/profile/        - Update user profile

TRIAGE & CHAT:
  POST /api/triage/         - Submit symptoms for analysis
  GET  /api/history/        - Get user's consultation history
  DELETE /api/history/{id}/ - Delete specific consultation

UTILITIES:
  GET  /api/places/         - Get place name from coordinates
  GET  /api/facilities/     - Get nearby health facilities
  GET  /api/stats/          - Application statistics

SETTINGS:
  GET  /api/translations/   - Multilingual text strings
  POST /api/voice-logs/     - Save text-to-speech logs

INSTALLATION AND DEPLOYMENT
================================================================================

BACKEND SETUP:
  1. Navigate to backend directory
  2. Create virtual environment: python -m venv venv
  3. Activate: source venv/bin/activate (Unix) or venv\Scripts\activate (Windows)
  4. Install dependencies: pip install -r requirements.txt
  5. Configure environment variables in .env file
  6. Run migrations: python manage.py migrate
  7. Start server: python manage.py runserver
  8. Production: gunicorn nepalcare.wsgi

FRONTEND SETUP:
  1. Navigate to frontend directory
  2. Install dependencies: npm install
  3. Configure API base URL in src/services/api.js
  4. Development: npm run dev
  5. Build: npm run build
  6. Production: Deploy build/ folder to static hosting


SECURITY CONSIDERATIONS
================================================================================

  • Token-based authentication (Bearer tokens)
  • Password hashing with Django's build-in authentication
  • CORS header validation for cross-origin requests
  • Environment variables for sensitive credentials
  • Input validation on both frontend and backend
  • SQL injection prevention via ORM (Django models)
  • HTTPS recommended for production deployment
  • User data encryption in transit and at rest
